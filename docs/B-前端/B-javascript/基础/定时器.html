定时的基本用法

            setTimeout(function(value){
                alert(value);
            },0,"hello")

例1

异步的定时器，先按顺序执行线程里的代码，异步程序加入队列。 队列里的函数必须等线程里的代码全执行完才会一个个按顺序执行。

            var timer1 = setTimeout(function(){
                console.log("timer1");
            },0)

            console.log(1);

            执行结果:
            1
            timer1

例2

线程里的代码没执行完将不会执行队列里的函数。

            var timer1 = setTimeout(function(){
                alert("timer1");
            },0);

            while(true){

            }

            定时器将永远不会执行。

例3

setTimeout是有顺序的先加入先执行。先进先出。

            var timer1 = setTimeout(function(){
                console.log("timer1");
            },0)
            var timer2 = setTimeout(function(){
                console.log("timer2");
            },0)
            var timer3 = setTimeout(function(){
                console.log("timer3");
            },0)

            执行结果
            timer1
            timer2
            timer3

例4

队列里前一个setTimeout定时器没执行完，下一个setTimeout定时器是不会执行的

            var timer1 = setTimeout(function(){
                while(true){

                }
            },0)
            var timer2 = setTimeout(function(){
                alert(2);
            },0)

            第二个定时器不会执行。

但是如果是事件呢？

执行事件函数时，会暂停队列中函数的执行。

            var oBtn = document.getElementById("btn");
              for(var i=0; i<10000; i++){
                  setTimeout(function(){
                      console.log("haha")
                  },i);
              }
            oBtn.onclick = function(){
                alert("ok");
            };
            //就算前面的10000个定时器没执行完也会，触发执行click的绑定函数。

            var oBtn = document.getElementById("btn");
            for(var i=0; i<10; i++){
                setTimeout(function(){
                    for(var i=0; i<10000; i++){
                        console.log(i);
                    }
                },i);
            }
            oBtn.onclick = function(){
                alert("ok");
            };

            //当一个定时器正在执行时，不会立即执行事件函数，
            //得等到这个定时器执行完才会执行事件函数，

例5

setInterval()不会退出队列，交替执行。

            var timer1 = setInterval(function(){
                console.log(1)
            },0);
            var timer2 = setInterval(function(){
                console.log(2)
            },0);
            var timer3 = setInterval(function(){
                console.log(3)
            },0)

            //交替执行三个定时器

例6

关于队列、队列执行环境、定时器延迟的问题。

            var timer1 = setInterval(function(){
                console.log(1)
            },50);
            var timer2 = setInterval(function(){
                console.log(2)
            },0);

            结果：222222222222 1 2222222222222 1 222222222222 1
            结果大致就像上述，不同情况下定时器2执行的次数稍微有变化。

             队列延迟执行
            这里定时器1在每隔50ms就插入队列，执行完后退出队列环境。一直循环。
            如果设置事件为0则可以认为是一直在队列环境，不会退出。


                定时器在队列的顺序右加入队列的先后顺序决定，
                但具体如何执行由队列顺序和延迟时间共同决定。


             队列：
             timer1;    50ms
             timer2;    0ms


             队列执行环境：

             1、将延迟时间为0的定时器按顺序加入队列执行环境
             timer2

             2、循环执行队列执行环境，此时只有timer2
            （此时没到调用timer1，所以执行环境就只有timer2）
             timer2

             3、当时间过了50ms后，timer1加入队列
             timer2
             timer1

             4、timer2执行完后， timer2在0m后加入队列执行环境
             timer1
             timer2

             5、队列重复上面的步骤。

例7

一个最简单的定时器嵌套。

            setTimeout(function(){                //timer0
                 console.log(0);
                 setInterval(function(){            //timer1
                     console.log(1);
                 },0);
            },0);

            结果： 01111111111111.。。。。。

            队列：
                1、初始
                     timer0;    0ms
                2、
                    timer1

            执行环境：

                1、将延迟时间为0的定时器按顺序加入队列执行环境
                        timer0;
                2、0执行完后 0退出  1加入队列并同时加入执行环境（这里1设置的没有延迟）
                        timer1;
                3、循环执行队列执行环境。
                结果：0111111111111111111111111

例8

             var oP = document.querySelectorAll('#out');
             setInterval(function(){        //timer0
                 oP[0].innerHTML += "timer0<br>";
                 setInterval(function(){    //timer1
                     oP[0].innerHTML += "timer1<br>";
                 },0);
             },0);

            队列：
                    timer0
            执行环境：
                    timer0
                    //由于0里面有定时器，所以在执行0时，会将定时器1加入队列位置在队列最后面

            队列：
                    timer0
                    timer1
            执行环境：
                    timer0
                    timer1

            队列：
                    timer0
                    timer1
                    timer1
            执行环境：
                    timer0
                    timer1
                    timer1
                    重复上面的步骤。。。。。。。。。
                    结果：001011011101111.。。。。。。。

                    嵌套的定时器会不停的有新的定时器加入队列

总结

  • 1、setTimeout() 只加入一次队列执行环境，执行完后，从队列执行环境删除。
  • 2、setInterval() 重复加入执行环境，执行环境在指定的时间及顺序执行，执行完后从队列执行环境删除。 什么时候加入执行环境？ 时间：设定的时间到了没？？ 顺序：执行环境是否阻塞？？ 时间到了，执行环境为阻塞，即可执行。
  • 3、队列只有一个，执行环境也只有一个。
  • 4、事件处理函数的执行优先级最高，前提是未阻塞。
  • 5、嵌套的定时器在每执行一次执行父定时器时子定时器加在队列最后面一次。 这就是为什么在时间处理函数内用定时器之前先清除定时器的原因。
